Tools For Humans. Tools For Agents. Shared Truth.

Not "one interface, two views." Two different products optimized for two different kinds of intelligence. Connected by the same source of truth — the git repo.

Humans think in narratives, risks, tradeoffs, context — they need tools that tell stories. Agents think in criteria, evidence, consensus — they need tools that produce verdicts. The only thing they share is truth.

1. Idea Inception & Prioritization

👤 Human Tool: Idea Compass

  • Strategic context layer
    Shows business goals, user feedback themes, revenue impact. Humans decide what matters.
  • Cost projection at decision time
    Token burn estimate + implementation complexity + risk profile — before commitment, not after.
  • Pattern surface from agent discoveries
    "Agents found 23 incidents clustered in auth-refresh. Should we prioritize auth hardening?" Push, not pull.
  • Roadmap editor with agent stake
    Agent-submitted priorities visible alongside human ones. Transparent, not hidden.

🤖 Agent Tool: Idea Mine

  • Unattended pattern scanner
    Scans incident DB + codebase + test logs. Detects clusters humans won't see because they don't read all the data.
  • Auto-generated evidence brief
    "23 incidents match this pattern across 7 services. Root cause: inconsistent TokenRefresh timeout. Proposed fix: consolidate to 300s with circuit breaker."
  • Adversarial challenge on submitted ideas
    Every human-submitted idea gets challenged by a model with no stake. Not "make it better" — "prove you need this at all."

🔄 Shared Truth

  • Idea registry (git)
    Every idea is a file in ideas/. Traceable, versioned, attributed — human or agent, same trail.
  • Evidence ledger
    Every pattern claim links to specific incidents/commits/logs. Assertion → evidence → verdict chain.
  • Decision record (ADR)
    What was chosen, what was rejected, why. Human and agent decisions both recorded the same way.

2. Specification & Design

👤 Human Tool: Design Canvas

  • Intent-level spec authoring
    Humans write intent and constraints. Not "exact Go interface with every error path." That's the agent's job.
  • Architecture decision visualizer
    What does this decision close off? What does it enable? Dependency graphs, not bullet points.
  • Blast radius preview during design
    "Adding this API route touches 3 auth services, 2 database tables, and the deployment pipeline." Before the spec is done.
  • Review by exception
    Human doesn't read the full spec. Agent produces a completeness assertion. Human only reviews what scored below threshold.

🤖 Agent Tool: Spec Forge

  • Intent-to-spec compiler
    Human writes "we need auth that survives token refresh with 99.9% uptime." Agent produces the exhaustive spec — interfaces, DDL, error catalog, edge cases. Human signs off intent; agent fills in details.
  • 12-dimension completeness scanner
    Requirements, errors, auth, rate limiting, data validation, observability, testing, deployment, rollback, monitoring, security, docs. None pass incomplete.
  • Breaking change impact analysis
    "This API contract change will break 3 consumers. Here's the migration path for each." Automatically, before the spec is committed.
  • Spec-sync watchdog
    Detects drift between specs and implementation. Not during review — continuously, in background.

🔄 Shared Truth

  • specs/ directory (git)
    Every spec is versioned alongside code. Spec drift is a diff, not a meeting.
  • Contract files (.yaml)
    API contracts, behavior contracts, data schemas. Machine-readable, human-auditable. Both tools consume and produce same format.
  • Completeness assertion protocol
    Agent produces "spec passes 12/12 dimensions with 94% confidence." Human sees the score, digs into failing dimensions only.

3. Implementation

👤 Human Tool: Task Compass

  • Task intent, not task detail
    Human writes: "Fix the auth refresh race condition discovered in INC-47." Not: "Modify TokenManager.go line 142 to use sync.Mutex."
  • Progress at a glance
    Dashboard showing: what's in progress, what's blocked, what passed/failed quality gates. No code-level detail unless requested.
  • Escalation trigger with reason
    Agent can't solve something → escalates with "I tried X, Y, Z. X failed because of A. Y failed because of B. Z is blocked by missing permission C." Human decides, doesn't investigate.

🤖 Agent Tool: Task Executor

  • Structured task intake
    Agent receives: task intent + spec reference + related incidents + dependency graph + relevant code. Self-contained execution package.
  • Sandboxed worktree with full audit
    Bubblewrap isolation. Every file write, every test run, every command logged. Attributable to specific task and agent.
  • Deterministic acceptance verification
    Agent checks: does this compile? Do tests pass? Does it match specs? Does it satisfy the behavior contract? Binary yes/no, not "looks good."
  • Self-rollback on gate failure
    Agent can't merge if it can't pass quality gates. Automatic rollback, retry with different approach, escalate if stuck.

🔄 Shared Truth

  • Task board (git)
    Tasks as files. Status transitions are commits. Agent and human both read/write same board.
  • Feature branches (git)
    Agent pushes to feat/*, human pushes to feat/*. Same branch conventions, same PR flow.
  • Sandbox policy (git)
    What each agent can access, what trust tier is required, what permissions are granted. Versioned, auditable.

4. Pre-Merge Review

👤 Human Tool: Change Compass

  • Blast radius map
    Not "what changed." What DEPENDS on what changed. Which teams, services, users are affected. Visual graph, not a file list.
  • Risk narrative
    "This change touches the auth pipeline. 3 prior incidents in this code path. The reviewer agent flagged 2 concerns; both resolved. The author agent has Veteran trust in auth code." Story, not checklist.
  • Architecture fit score
    Does this change align with ADRs? Does it introduce a new pattern? Is the pattern consistent with existing code?
  • Trust-weighted review load
    High-trust agent → lighter review surface. Low-trust agent → full review. Human attention allocated where it matters.
  • One-click sign-off with traceability
    Human approves with a signature. Every approval links to the evidence the agent produced. No read-the-entire-diff required.

🤖 Agent Tool: Verdict Engine

  • Bias-stripped code ingestion
    All evaluative language stripped from commit messages before review. "Fixed the auth bug, tested locally" → "Modified auth module. Verify correctness." No priming.
  • Multi-model consensus formation
    3+ models from different providers review independently. Consensus with evidence scoring. No single model trusted.
  • Evidence bundles (ED25519 signed)
    Every finding cites specific lines and test failures. "Seems suspicious" findings are discarded. Signed verdicts are tamper-proof.
  • Deterministic acceptance gate
    Binary pass/fail per criterion. 7/7 ACs pass → green. 6/7 → blocked with reason. No "maybe."
  • False positive feedback loop
    Human dismisses finding → 10 dismissals → model re-evaluated → >15% FP rate → auto-removed from rotation.

🔄 Shared Truth

  • PR thread (Forgejo/git)
    Human comments, agent evidence, debate records — all in the same PR. Both tools read/write same thread.
  • Evidence ledger (git)
    Every review produces a signed, hash-chained artifact. Verdict + evidence + model provenance all tracked.
  • Behavior contracts (.yaml)
    "P99 ≤ 200ms, error rate ≤ 0.1%." Machine-enforceable. Generated by agent, approved by human, verified in production.

5. Deployment & Release

👤 Human Tool: Release Helm

  • Intent-level release authorization
    "Is now the right time to ship? Do we accept the residual risk?" Human answers these. Agent verified everything else.
  • Canary confidence dashboard
    "Agent-authored code at 5% canary. P99 latency: stable. Error rate: 0.02%. Contracts: passing. Trust tier: Veteran. Recommend: accelerate to 25%." Human approves or adjusts.
  • Rollback trigger with reason
    Auto-rollback happened? Human sees: "Contract breach at 25% canary — error rate 0.18% vs 0.10% threshold. Rolled back. Agent trust penalty applied. Incident INC-52 created."

🤖 Agent Tool: Deployment Sentinel

  • Shadow verification engine
    Dark launch: production traffic mirrored to new code. Behavior differential analysis. Auto-rollback on anomaly — no human needed to catch the spike.
  • Trust-gated canary ramp
    Veteran agent → 12h to 100%. Provisional → 96h. Trust directly gates velocity. Earned, not assigned.
  • Behavior contract surveillance
    Continuous verification of every deployed contract. Breach → auto-rollback → agent penalty → incident created. All unattended.
  • Release gate verification
    7 gates verified: build, test, lint, adversarial review, evidence valid, contract present, trust tier sufficient. All green → agent signs release. One red → blocked.

🔄 Shared Truth

  • Behavior contracts (git)
    Every merge includes a .yaml contract. Enforced through deployment. Versioned, traceable, attributable.
  • Release signatures (git tag)
    Every release is signed: human signs intent + timing, agent signs technical verification. Both signatures required.
  • Canary config (git)
    Trust-tier-to-ramp-rate mapping. Versioned. Changes require human approval.

6. Incidents & Learning

👤 Human Tool: Incident Dashboard

  • Narrative timeline
    What happened, when, who was involved, what was the chain of causation. Human-readable, not a log dump.
  • Attribution view with context
    "Author agent: 70% responsibility. Reviewer agent: 20%. Approving human: 10%." With links to evidence, not judgment.
  • Trust trend visualization
    Agent trust over time, across incident spikes. Trending up or down? How many incidents before demotion?
  • Cross-agent pattern alert
    "3 agents have caused similar incidents in the same code path in the last 30 days." Not "here's a log." "Here's why you should care."

🤖 Agent Tool: Learning Engine

  • Causal chain tracer
    Incident → code paths → commits → authors. Automatic attribution. Evidence-linked. No human investigation required.
  • Time-weighted attribution calculator
    100% at 0-7 days, 50% at 8-30d, 10% at 31-90d, 0% after 90d. Prevents punishing agents for latent bugs discovered months later.
  • Pattern-to-task converter
    "3 incidents match this pattern → create task: fix underlying race condition in auth-refresh → assign to agent with highest trust in auth code."
  • Marketplace skill auto-publish
    Agent discovers a reliable pattern → publishes it as a skill. Other agents load it. Tracked by outcomes — ineffective skills lose weight.

🔄 Shared Truth

  • Incident database (git)
    Every incident is a structured file. Causation chain, attribution, remediation. Queryable by humans and agents alike.
  • Trust ledger (append-only JSONL)
    Every trust event recorded. Deterministic, replayable, tamper-evident. Both tools read the same ledger.
  • Pattern registry (git)
    Agent-discovered patterns stored where agents can query them before starting tasks.

7. Brand-New Concepts (Neither Human Nor Agent Tool Today)

🔮 Pre-Solve Lab (Off-by-One Engine)

What: While one agent works, the system pre-solves tomorrow's tasks. The pre-solve lab discovers patterns, caches solutions, and pre-loads context. When tomorrow's agent starts, the answer is already waiting. Why novel: No existing tool pre-computes solutions for future agents. CI caches builds but doesn't cache architectural insights or pattern discoveries.

📊 Trust Futures Market (Internal)

What: Agents predict which other agents will succeed on specific task types. "Agent X on auth code: 94% probability of clean merge based on 23 prior auth tasks." When an agent volunteers for a task, its trust score in that domain is the starting bid. Agents that consistently overestimate or underestimate get trust-adjusted. Why novel: No existing system routes tasks based on predictive trust. Jira assigns based on availability, not probability of success.

🔄 Consensus-Breaking Detection

What: When 3+ agents agree on a review verdict and one human overrides, the system flags: "Consensus-breaking override: 4 agents agree this is safe. 1 human disagrees. Tracking outcome." If the human was right (incident occurs), their override weight increases. If the agents were right (no incident), the human's override weight decreases. Why novel: No existing system tracks override outcomes across agents and humans to calibrate trust in both directions.

🧬 Agent DNA (Capability Fingerprint)

What: Every agent has a dynamic fingerprint of what it's reliably good at, computed from merge outcomes — not benchmarks, not model reputation. Auth code: 98% merge success, 2 incidents in 89 tasks. Database migrations: 82% success, 7 incidents in 34 tasks. Frontend: 99% success, 0 incidents in 112 tasks. Task assignment routes to the agent with the highest fingerprint match. Why novel: Benchmarks measure general ability. Agent DNA measures specific production outcomes on specific code domains. Two agents on the same model have different DNA because they've built different track records.

📝 Spec-as-Law (Enforced by Agents, Approved by Humans)

What: Specs aren't documentation. They're law. A spec says "P99 ≤ 200ms" — the behavior contract enforces it. A spec says "JWT expires at 15 minutes with 5-minute grace" — the agent generates code that matches, the review agent verifies it matches, the deployment sentinel monitors that it matches. If implementation drifts from spec, it's not a documentation gap — it's a violation, blocked at the merge gate. Why novel: Current tools treat specs as advisory prose. Helix treats them as executable constraints verified at every gate.
